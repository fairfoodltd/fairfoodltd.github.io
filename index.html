FairFood - Track Food from farm to plate with NFC blockchain


This project is an Android application that transparently tracks any kind of product during its supply chain lifecycle.

How it works?

A manufacturer attaches a NFC tag on his product and registers it on the Ethereum blockchain network by only pressing 3 buttons within the app. Any time a new agent/entity interacts with the product, he uses the app to scan the attached NFC tag to get the product's ownership (with only 1 button press) before he physically takes the product.
This continues until the end buyer consumes the product (e.g. eats the food product, attaches the new tire to his car). All those transfers of ownership are recorded on the blockchain network which is immutable, and they are visible in real time within the app that any potential customers, resellers, or goverment bodies can use.

Technologies benefits:

Blockchain technology allows instant automation of transactions and product ownership transfer and validation between any entities in a supply chain. Its applications benefit from the immutability and transparency for any piece of data that is put on the blockchain.

NFC tags are tamper-proof and can store encrypted information. They can be attached on or inside a product’s package to store its details and prevent altering the object’s authenticity.

This tech incorporated in a common smartphone would easily grant any customer the ability to get informed about all the characteristics of a product, trace its journey prior to receiving it and check its manufacturer legitimacy. It would also allow producers to showcase their products nicely to possible customers and set them ready to be admitted on the supply chain.

Migrating automatically data from Google Calendar to Excel



I have done this project during my summer internship at Technocamps. At the end of the project I have achieved the desired result and my manager praised my effort for this original solution that increased their operational efficiency (recommendation provided below).

As mentioned before: This project runs a script automatically that extracts and serializes all data from Tehcnocamps' shared Google calendar and deserializes it on the local server as custom Excel sheets. The goal was to extract and back up this data in a custom tabular human readable format, that the administrative team and HR could easily use for legal and booking activities.

I was running the script on an Ubuntu virtual machine that I have installed on the Technocamps servers, using Cron automation (an ubuntu/linux tool). The script uses Google OAuth to get access to the shared calendar and Google Calendar API to request the data that gets serialized as Json. Afterwards, desirialize that data using the documentation available and put it in the custom built Excel sheets that I have done using XlsxWriter library.

For more information check the README on Github. I didn't put a video as I don't want to disclose any sensitive information.

The project was build in Python3 using Google Calendar API, Google OAuth, Json, Cron automation, an Ubuntu VM and XlsxWriter library.

A blockchain system built in Java



The video presents a short indroduction to blockchain ledgers.

This blockchain system features the following: creation, recording and signing of transactions and wallet addresses with public/private keys pairs, securing transactions using Elliptic Curve Digital Signature Algorithm (ECDSA) encryption algorithm and verifying transaction authenticity.

Missing functionallity from a production ready blockchain: nodes connecting and mining the network, a RESTful API and smart contracts compatibility. Part of the code was forked from project nayuky that handled the mathematics behind ECDSA and conversion between binary data and ASCII strings.

This was fully coded in Java using integrated cryptography libraries with the goal of implementing a blockchain similar to Bitcoin. For proof and test cases of the system in action, check the Github repo documentation.
